# Product Hunt demo script

**Target:** 45–60 seconds. Record a clean Claude Code installation in real
time. Use a fresh disposable workspace and do not expose private business data.

## Shot list

| Time | Screen | Recording direction |
|---:|---|---|
| 0–06s | Landing outcome | Hold on “Know what matters today.” |
| 06–14s | Claude Code | Type the two public install commands. |
| 14–38s | Onboarding | Run `/founder-os-init`; show Business, Customer, Quarter, and Money progress labels. |
| 38–52s | Saved brief | Open the real `reviews/daily/YYYY-MM-DD.md` produced by that run. |
| 52–60s | Source trace | Follow its one thing to the owned queue item and quarterly bet. |

## Commands shown

```text
/plugin marketplace add msolecki/founder-os
/plugin install founder-os@founder-os
/founder-os-init
```

## Spoken script

Founder OS helps a company of one decide what matters today before the inbox
makes that decision for them. Install the public Claude Code plugin, then run
Founder OS init once. The flow asks four short groups: Business, Customer,
Quarter, and Money. Unknown numbers stay unknown; they become follow-ups instead
of confident fiction. Each answer goes to the role that owns its Markdown file.
Setup ends only when the same workspace contains a valid
reviews/daily/YYYY-MM-DD.md with one priority and one explicit trade. Here, the
brief selects an existing queue commitment, ties it to a quarterly bet, and
shows the source behind the choice. The files stay on your machine. Founder OS
never sends messages or moves money; it drafts, records, and challenges, while
you make the decision. It is free and MIT-licensed. Try the activation flow and
tell me the first place it breaks.

## Recording integrity

There are **no hidden cuts** across installation, onboarding, validation, or
brief persistence. If the real run fails, keep the failure visible and record a
new complete run only after the defect is fixed and tested. Never splice a
Studio North example onto a different run and present it as that run's output.
