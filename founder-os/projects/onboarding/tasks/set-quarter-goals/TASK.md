Run `quarterly-planning`. The bets for this quarter, each with an outcome and a
kill condition.

A bet without a kill condition is a hope. Write the kill condition now, while
you are not yet emotionally invested in the bet.

The bets get written here, in this session, from the charter and the ICP the
last two steps produced — not handed to you as a planning exercise for the
weekend.

**This is `quarterly-planning`'s first run, and the skill has a branch for it.**
Read it there: no `reviews/quarterly/` and no bets in `goals.md` means step 1
(verdict last quarter) and step 7 (`red-team`) are skipped and said out loud, and
"never mid-quarter" does not apply — it governs reopening a plan, and there is
nothing to reopen. You almost certainly installed mid-quarter; almost everyone
does. Size the bets to the weeks that are actually left, not to thirteen.

The first plan ships unattacked. That is a debt, not a rule bent: the Board
Member is a session and this task has minutes. Next quarter it gets red-teamed.

This is also the file tomorrow morning runs on. `daily-brief` names one thing a
day and ties it to a bet in `goals.md`; a quarter with no bets in that file
produces a brief that can only agree with whatever you already intended to do.

Anything a bet needs from you that is not a decision — a number, a conversation,
a document — goes to the **Chief of Staff** for `queue.md`, with an id and this
bet's name on it.
